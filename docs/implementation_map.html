Implementation Map

Target paper: High redshift cosmography: new results and implication for dark energy (Demianski et al., 2012).

This document is designed to be the bridge between the paper and a faithful reproduction notebook. It prioritises chain-of-custody: first recreate what the paper actually did, then isolate later changes in data or method.

1) Scope and success criteria

Primary goal

Reproduce the paper’s baseline cosmographic analysis as faithfully as practical before introducing any 2026 updates.

Not the goal yet

Do not improve the method, refresh priors, replace datasets, or “fix” cosmography in the baseline notebook.

Success condition

Recover the paper’s main tables, key posterior behaviour, and headline qualitative claims within documented tolerance.

Baseline success criteria

  • Recreate the paper’s luminosity-distance series and Hubble-series machinery.
  • Implement both z and y = z/(1+z) parameterisations.
  • Reproduce “Cosmography I” and “Cosmography II”.
  • Recover Table 1 and Table 2 to within reasonable numerical tolerance.
  • Recover qualitative statements: negative q0, weakly constrained s0, unconstrained l0, faster chain convergence in y, and tension in j0 versus ΛCDM.

3) Equation-to-code map

3.1 Cosmographic state variables

Implement the paper’s definitions directly, with present-day parameters H0, q0, j0, s0, l0.

H(t) = (1/a) da/dt
q(t) = -(1/a) (d²a/dt²) / H²
j(t) = +(1/a) (d³a/dt³) / H³
s(t) = +(1/a) (d⁴a/dt⁴) / H⁴
l(t) = +(1/a) (d⁵a/dt⁵) / H⁵

Paper sections: Eqs. (2)–(10).

3.4 Recommended unit tests

  • dL(z→0) ~ cz/H0.
  • mu_theory monotonic in z over physical ranges used.
  • H(z)^2 > 0 and dL(z) > 0 for allowed samples, matching the paper’s positivity cuts.
  • z and y implementations agree at low redshift.
  • Series coefficient unit tests copied directly from the paper equations.

5.3 MCMC settings to mirror

  • Paper says: two times four independent chains, about 100000 elements each.
  • Convergence check: Gelman-Rubin style criterion with R - 1 ~ 0.01.
  • Discard first 30% as burn-in.
  • Thin chains before coadding.

Implementation note: modern libraries can do this more cleanly, but baseline mode should keep these choices visible and documented.

7) Ambiguities and traps

7.1 Known ambiguity list

  • Exact data products: the paper names datasets, but not every machine-readable file path.
  • GRB calibration provenance: reproducing Cosmography II depends on recovering the same calibrated GRB HD, not merely “some” Amati sample.
  • CMB shift parameter use: this is baked into the baseline, but later should be treated as methodologically fragile in a pure cosmographic context.
  • Order language: the paper speaks of “fifth order” but the observable series implementations can look like fourth-order in z after factoring out the leading term; document this carefully.
  • Flatness assumption: the paper says spatially flat models only for the working analysis; baseline code should expose this explicitly.
  • Positive-definiteness cuts: these are not a tiny detail; they shape the sampled parameter volume.

7.2 Things likely to bite during implementation

IssueConsequenceMitigation
Mixing nuisance parameter handling for SN with later modern covariance stylesSilent mismatch in χ²Keep baseline SN block isolated
Using updated h or matter priors too earlyMoves posteriors before you know whyFreeze all legacy prior numbers in baseline mode
Implementing only one of z/y and inferring the otherConflates parameterisation and numericsBuild both paths explicitly
Substituting modern MCMC defaultsCan alter effective posterior summariesKeep old-style settings available, even if slow
Skipping GRB sample archaeologyCosmography II will not be faithfulMark Cosmography II as provisional until source matched

9) Execution plan

Step 1

Freeze the legacy assumptions and numbers into a config file.

Step 2

Implement series expansions and low-level unit tests.

Step 3

Implement SN-only baseline and verify distance-modulus behaviour.

Step 4

Add BAO, shift parameter, and H(z); reproduce Cosmography I.

Step 5

Recover the calibrated GRB HD and reproduce Cosmography II.

Step 6

Only after that, branch into 2026 data and method updates.
